JavaScript Use Strict

  • What: literal expression
  • Why:
      Write "secure" JavaScript
      Makes "bad syntax" that regular JS ignores into real errors.
      Can't accidentally create global variables with mistyping

Strict Mode will throw errors for the following:

  • non-writable property
  • getter-only property
  • non-existing property
  • non-existing object

Examples of practices that are NOT allowed in Strict Mode

  • Using a variable w/o declaring it
  • Using an object w/o declaring it
  • Deleting a variable (or object): delete x;
  • deleting a function
  • Writing a read-only/get-only property

    • Plus much more!
